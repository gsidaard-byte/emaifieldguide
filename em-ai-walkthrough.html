The idea
Why this guide exists

The whole idea in three steps

Entrepreneurial mindset is made from 18 habits, each formed through repeated practice. AI can deepen that practice or quietly erase it. The 18 habits of entrepreneurial mindset are built, not taught. AI is now the biggest force acting on how they get built.

What this is A working guide to practicing 18 entrepreneurial mindset habits in an age of AI, for your own work and your classroom.
How to use it Read the idea below, then pick a habit that matters to you. Each one shows how AI helps, the trap to avoid, a prompt to run yourself, and an activity to run with students.
What you'll leave with One habit worth practicing this week, a prompt you can use today, and a class activity you could run next.
1

Habits form through practice

Curiosity, systems thinking, value creation. None of these are facts you can teach in a lecture. They are habits, and habits are built by doing the thinking yourself, repeatedly.

2

AI short-circuits practice by default

Every student and every faculty member now has AI in their pocket. Used naively, it hands over clean answers and the thinking never happens. Each habit has a specific failure mode, an anti-pattern, where AI quietly does the habit for you.

3

The right prompt flips the relationship

Each prompt in this guide turns AI into a sparring partner for one specific habit. Instead of AI being curious for you, the prompt makes you practice being curious. Every habit comes in two modes. Run it yourself in your own work, or run it with students as a classroom activity.

The master principle

You can't evaluate work in a domain you haven't mastered, so polished output reads as competent when it isn't. AI amplifies the judgment you bring. It cannot supply the judgment you lack. Develop expertise first, then let AI scale it. Never the reverse.

Research basis
The evidence behind the anti-patterns

Research basis

The 18 habits come from the KEEN Habits of Entrepreneurial Mindset. The claims that need evidence are the anti-patterns, the specific ways careless AI use erodes each habit. These studies are the anchors.

A note on coverage. Seven habits (Opportunity Seeking, Future-Minded, Systems Thinking, Customer-Centric Thinking, Impact Thinking, Socially Minded, and Resourcefulness) have no study matching their specific anti-pattern. The offloading and homogenization mechanisms above plausibly generalize to them, but habit-specific evidence is an open research gap. We cite where the evidence is and stay quiet where it isn't.

Use this guide live, inside your assistant

Everything in this guide is also a live tool. Connect EMxAI Studio once and the habits, prompts, and workflows become available on demand inside Claude or ChatGPT, always current, nothing to copy. The server is public, read-only, and stores nothing.

New to MCP? Here's the short version

The Model Context Protocol is an open standard that lets an AI assistant connect to an outside server and pull in extra capabilities. Instead of pasting a long prompt every time, you connect this server once and the curated prompts appear inside the assistant, ready on demand.

✓ Read-only and stateless. It only ever returns text, prompts and reference content.
✓ No login, no account, nothing to install beyond one config line.
✓ Stores no data and cannot modify files, send messages, or act on your behalf.

Connect in Claude

Add this to your claude_desktop_config.json, then fully quit and reopen Claude. The prompts appear in the slash menu, type / to see them.

{
  "mcpServers": {
    "emxai-studio": {
      "type": "http",
      "url": "https://emxai-studio-mcp-production.up.railway.app/mcp"
    }
  }
}

Then try /em-list to see the whole map, or /em-diagnostic to get routed.

Connect in ChatGPT

Go to Settings, then Connectors, then Add MCP Server. Paste the endpoint URL, choose No Auth, and confirm. The capabilities appear as tools the assistant calls when asked.

https://emxai-studio-mcp-production.up.railway.app/mcp

Then ask, for example, use the em_habit tool for Systems Thinking.
